2. Justificación de Fases

Fase 1: Escalabilidad (Python vs R)

La transición a una tubería 100% nativa en Python fue forzada por el tamaño del dataset (>4M células). La fricción de conversión entre objetos AnnData y Seurat consumía recursos críticos y generaba inconsistencias. Python permitió una gestión eficiente de los 80GB de datos brutos.

Fase 2: Estrategia de scAR (Nivel Donante)

Debido al límite de 64GB de RAM, la remoción de RNA ambiental se realizó a nivel de donante. Esta decisión técnica no solo salvó el proceso de fallos de memoria, sino que aumentó la precisión biológica al modelar el ruido ambiental de forma local para cada pozo de secuenciación.

Fase 3: El Filtrado Declarativo (V2-Clean)

Para garantizar una señal NK pura, se eliminaron preventivamente genes de Inmunoglobulinas y receptores T. Esto aseguró que la firma de envejecimiento no estuviera contaminada por transcritos ambientales de poblaciones altamente productivas (B y T cells).

Fase 4: El Rigor Estadístico (V3-Perfect)

Con un n=547 donantes, el poder estadístico detectaba ruidos insignificantes como significativos. La implementación de LFC Shrinkage permitió colapsar estos falsos positivos hacia cero, dejando una firma robusta de 182 genes defendible científicamente.

Fase 5: Alineamiento Científico (V4-Clean)

Se removieron los 45 genes ribosomales para evitar el sesgo dominante en los análisis de vías genéticas. Esto permite que el reporte de enriquecimiento muestre los mecanismos biológicos subyacentes más allá de la maquinaria de traducción.
